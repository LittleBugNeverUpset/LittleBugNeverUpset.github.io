对称加密（Symmetric Ciphers）

密码

一个密码由三元组 (𝒦, ℳ, 𝒞) 和二元组 (ℰ, 𝒟) 组成。其中，𝒦 为密钥空间、ℳ 为明文空间、𝒞 为密文空间、ℰ 为加密算法、𝒟 为解密算法：

ℰ : 𝒦 × ℳ → 𝒞，𝒟 : 𝒦 × 𝒞 → ℳ

且满足:

∀m ∈ ℳ, k ∈ 𝒦 : 𝒟(k, ℰ(k, m)) = m

其中，ℰ 常为随机算法，而 𝒟 常为确定算法。对称加密相较于普通密码多一个密钥生成算法，用于生成密钥

1. 一次一密（One-Time Pad，OTP）

对于一次一密，加密过程非常简单：取明文m ∈ {0, 1}n，密钥k ∈ {0, 1}n，其中密钥真正均匀随机，且只使用一次，密文m = c ⊕ k。直觉上讲：当你拿到一个密文c时，由于密钥k是完全随机的且只有你和对方知道，所有可能的明文m′都同样有可能。换句话说，对于攻击者而言，密文c不提供任何关于明文的统计信息——它可以是任何东西的密文。

香农给出了完美保密的定义：如果对于任意两个明文m1, m2 （长度相同）以及任意密文c有：

其中概率取自密钥K的随机分布（以及加密算法的随机性，但 OTP 加密是确定性的）。等价表述：给定密文c，明文的后验分布等于其先验分布（即密文不提供任何信息）。

证明（暂略）

结论：一次一密是信息论意义上绝对安全的。即使攻击者拥有无限计算能力，也无法从密文中获得任何关于明文的额外信息（除了明文长度——这是所有加密方案都无法隐藏的，除非特殊处理）。

1.1 OTP 的致命局限（为什么实际很少用）

虽然理论完美，但 OTP 有严格的约束，导致它几乎只用于极少数场景（如外交、军事热线）： 1. 密钥长度等于明文长度。如果要有能力加密任意长的消息，你需要预先共享同样长的密钥。这带来巨大的密钥管理负担。 2. 密钥必须真正随机，不能是伪随机数（如 C 语言 rand() 或普通 PRG）。需要物理真随机源（抛硬币、量子随机数等）。 3. 每个密钥只能使用一次。重复使用密钥会彻底破坏安全性 4. 需要安全的密钥分发渠道。既然密钥和明文一样长，分发密钥等同于分发明文本身，只是时间上提前了。

1.2常见误解澄清

“一次一密不安全，因为可以通过穷举密钥得到所有明文”：穷举攻击确实能得到所有可能的明文，但攻击者无法判断哪个是正确的——所有明文等可能。这与“安全”并不矛盾，因为安全要求的是“无法获取信息”，而不是“无法枚举可能性”。

“OTP 无法抵抗密文篡改”“对，OTP 只提供保密性，不提供完整性或认证。攻击者可以翻转密文中某一位，导致解密后明文的对应位也翻转，且接收方无法察觉。所以实际系统必须结合 MAC 或使用认证加密（如 OTP+MAC 就是 Carter–Wegman 结构）。

2. 流密码（Stream Ciphers）

流密码的诞生思想来自于试图模拟OTP的效果，但用短的种子（例如 128 比特）通过伪随机生成器（PRG）扩展成长密钥流。这样密钥管理简单，但安全性降级为计算安全：如果 PRG 是安全的，则流密码在计算能力有限的敌手下看起来像OTP。 OTP是衡量流密码安全性的理想标准 —— 如果一个流密码的密钥流不可与真随机序列区分（在多时间敌手下），那么它本质上就是实用的OTP。

2.1 PRG的不可预测性 (Unpredictability)

一个安全的PRG必须是“不可预测的”。这意味着，即使攻击者知道某个PRG输出序列的前i位，在计算上，他也无法以不可忽略的优势正确地猜出第i+1位。反之，如果能被预测，那么这个PRG就是不安全的。“一次一密”虽然完美安全，但密钥必须和消息一样长，非常不实用。因此引入PRG，用一个短的“种子”密钥生成长的伪随机密钥流ℰ(k, m) = m ⊕ 𝒢(k)。牺牲理论上的“完美安全”，换取计算上的“语义安全”，这依赖于PRG在计算上是不可预测的

一个PRG如果是在计算上不可区分的（Computational Indistinguishability），那么对于任何在多项式时间内运行的敌手，伪随机序列和真随机序列是“看起来一样”的。如果PRG是安全的，那么由其构建的流密码就具有语义安全性（Semantic Security），即敌手无法从密文中获取任何关于明文的信息。

如何量化一个PRG是否安全？这里的关键在于“优势”（Advantage）的概念。它是一个统计测试（Statistical Test）的输出结果与随机猜测之间差值的绝对值，用于衡量敌手成功区分伪随机序列和真随机序列的能力。如果这个优势是“可忽略的”（Negligible），我们就可以认为该PRG在计算上是安全的。

可忽略（Negligible）

在理论密码学中，“可忽略”是一个形式化的数学定义，用于在安全证明中精确量化一个事件发生的概率。

可忽略函数：一个函数 ε(n)如果最终变得比任何多项式函数的倒数都小，那它就是可忽略的。更严谨地说：对于任意多项式 p(n)，都存在一个数 N，使得所有n > N 都满足 ε(n) < 1/p(n)。

不可忽略函数：如果一个函数不是“可忽略”的，它就是“不可忽略”的。这意味着存在某个多项式 p(n)，使得 ε(n) ≥ 1/p(n) 的关系对无穷多个n的值成立。

这里用多项式来定义是有原因的。因为密码学中的“高效”通常意味着“多项式时间”（Polynomial Time），因此“可忽略”就像定义了一个在多项式时间攻击下的“安全边界”。

作为安全边界的阈值：如果攻击者的“优势”（Advantage）是可忽略的，我们就可以认为这个方案是安全的。

2.2 脆弱的PRG示例

线性同余生成器（Linear Congruential Generator, LCG）：这是课程中经典的脆弱PRG案例，一些统计学库（如glibc）中曾经使用过。它的公式如下：

r[i] ← (a * r[i − 1] + b) mod  p

尽管其输出在简单的“0和1个数统计”这类测试中可能表现良好，但它有一个致命的弱点：可预测性。这是因为LCG的输出是其内部状态的线性函数，只要收集到少量连续输出，攻击者就能轻松求解出参数和内部状态，从而预测后续所有输出。

rand() / random() 函数： 永远不要使用C标准库中的rand()或random()等函数来生成密钥或用于任何密码学目的。这些函数设计的初衷是为了统计学上的随机性，而非抗预测性。它们的输出往往具有固定的模式或周期，对于抵御精心设计的攻击是不安全的

2.3 实践中应当避免的错误

重复使用密钥（Two-Time Pad）：这是最常见的错误，也是最致命的。如果在不同的消息中使用了相同的密钥流（即Two-Time Pad）攻击者只需将两个密文进行异或，便可得到两个明文异或的结果：

c1 ⊕ c2 = (m1 ⊕ 𝒢(k)) ⊕ (m2 ⊕ 𝒢(k)) = m1 ⊕ m2

由于自然语言（如英语）或文件格式（如PDF、ZIP）存在大量冗余信息，攻击者很容易从中分析出原始明文，例如，历史上著名的“维诺那计划”（Venona Project，1941-1946）正是利用了两次一密（Two-Time Pad）这一弱点，破解了苏联的间谍通信。

使用任何可预测的PRG，都相当于从根本上摧毁了加密系统的安全性。攻击者一旦能根据已知信息预测出密钥流，就可以任意解密或篡改后续的通信内容。安全的PRG通常是确定性的（同样的种子永远产生同样的密钥流）。但在网络通信中，为了使用同一个长期密钥加密海量数据包，我们必须引入一个公开的、必须永不重复的Nonce（Number used once）或IV（Initialization Vector），并与长期密钥组合作为PRG的输入，以确保每个数据包都使用不同的密钥流

802.11b WEP的经典失败案例：WEP协议使用了24位的IV。在繁忙的网络中，这个长度过短的IV很快就会被重复使用，从而导致Two-Time Pad攻击。更糟糕的是，某些实现中，IV会在设备重启后重置为0，这极大地加速了密钥流的重用，使攻击变得轻而易举。

流密码的延展性 (Malleability)：流密码（包括OTP）和大多数仅提供保密性的加密方案一样，不提供完整性保护。可延展性指的是攻击者可以在不知道密钥和明文的情况下，通过有目的地修改密文，来“控制”解密后的明文内容。因为加解密都只是简单的异或运算，这个性质是对OTP与生俱来的数学特性的直接利用。

整个攻击过程可以拆解为以下几个步骤：

发起攻击：攻击者拦截到原始密文c，虽然无法解密，但他可以选择一个任意的值p，并计算出新的密文c′ = c ⊕ p发送给接收方。

接收方解密：接收方用共享的密钥k解密新的密文c′:

达成效果：接收方最终看到的是被篡改后的明文m′ = m ⊕ p 整个过程不需要知道密钥k，最终解密出的结果会精确地按照攻击者设定的模式p发生改变。

总结：我们追求的是一种在理论上完美安全（Perfect Secrecy）但不太实用的OTP（一次一密）的特性。为了实用化，我们用PRG来模拟它。一个安全的PRG必须满足不可预测性，而线性同余生成器或rand()这类统计上看起来“随机”的算法，实际上却非常脆弱和可预测。因此，在实践中必须彻底避免使用它们。同时，必须严格遵守绝不重复使用密钥流的铁律，并小心处理Nonce和IV的生成以及数据完整性保护的缺乏问题。

3. 流密码的实际应用

3.1 RC4

RC4是由Ron Rivest在1987年为RSA Security设计的流密码，它以极简、极快著称，完全基于字节操作，没有复杂的数学运算。其本质上是一个伪随机数生成器（PRNG），它将一个可变长度的密钥（通常40~256位）扩展成一个密钥流（keystream），然后将密钥流与明文逐字节异或得到密文。整个算法分成两个阶段：

RC4 密钥调度算法

目标：用用户提供的密钥（通常为5~32字节）初始化一个256字节的状态数组S。经过256次置换后，S数组被“打乱”，但仍然是一个0~255的排列。

RC4 伪随机生成算法

目标：从当前状态数组S中不断产生密钥流字节。输出的字节序列就是密钥流，与明文异或得到密文；解密时完全相同（因为RC4是对称的）。

直观理解：RC4利用状态数组S（一个排列）和两个指针不断“随机漫步”，生成看似无规律的输出。

RC4的弱点

从信息论和统计分析角度来看RC4存在多个系统性缺陷，这些缺陷使得RC4无法提供真正的随机性，从而被现代密码学淘汰。

密钥扩展算法（KSA）的根本性缺陷

弱密钥（Weak Keys）问题：某些密钥模式会导致初始S数组在KSA之后依然存在可预测的结构。当密钥长度是2的幂（例如16字节）时，某些密钥能使S中很多位置的数值与索引之间存在线性关系。攻击者只需知道密钥的一小部分比特，就可能推演出大量输出字节。

关键字节的统计相关性：KSA并不是一个完美的打乱过程，它会导致初始S数组的第一个字节S[1] 与密钥的第一个字节之间存在强相关性。这种相关性会传递到PRGA的输出，尤其是前几个字节。

输出字节的统计偏差（Statistical Biases）

这是RC4最致命的弱点：其输出的概率分布不是均匀的，存在多个可观测的偏差。

Z2 = 0 的双倍概率偏差（Mantin & Shamir, 2001）：RC4输出流的第二个字节Z2等于0的概率约为2/256 = 1/128。真随机序列中任意字节等于0的概率为1/256。偏差倍数为2倍，且该偏差不会消失——即使输出到很远的字节，这个偏差依然存在（只是主要影响前若干个字节）。仅需观测约O(N)个输出（N=256），就能以极高置信度区分RC4密钥流和真随机数。

Z1的负偏差与正弦分布（Mironov, 2002）：RC4的第一个输出字节Z1等于0的概率低于1/256（约1/260），且所有256个可能值的概率呈正弦波形。该偏差的理论证明直到2010年左右才完成。

更多字节的“偏向零”偏差：后续研究发现：Z3 到 Z255 以及更后面的许多字节，都存在不同程度的偏向0的偏差。甚至在整个RC4长周期中，偏差会周期性地重现。

可证明的区分攻击（Distinguishing Attack）

由于上述统计偏差，存在一个简单的区分器：收集RC4的前256字节输出，统计其中0的个数是否显著高于理想值。若显著偏高，判定为RC4。这种攻击所需的样本极少（几千个字节），而完美的密码本应需要天文数字的样本才能区分。

针对实际协议的实战攻击

破坏WEP协议（Fluhrer, Mantin, Shamir, 2001 - FMS攻击）：WEP将24位的初始向量（IV） 与固定密钥拼接成RC4的密钥。即Key = IV || 固定密钥。攻击者可以主动或被动的收集大量使用不同IV的加密数据包。由于IV是明文传输，攻击者能够构造许多“相关密钥”。利用KSA的弱密钥 + 第一个字节的偏差 → 可以从统计中恢复出固定密钥（40位或104位）。仅需要约400万~600万个数据包，几秒内即可破解。

威胁TLS协议（2013年）：研究人员发现：在TLS会话中如果使用RC4，攻击者可以通过注入大量会话（cookie多次重复加密）并利用输出字节偏差，直接恢复出加密的cookie。这导致浏览器厂商和IETF正式弃用RC4。

3.2 内容扰乱系统(Content Scrambling System，CSS)

“混淆系统CSS”——著名但已失效的DVD加密系统，其核心教训是：一个仅仅基于线性反馈移位寄存器（LFSR）且密钥长度受限的密码是多么脆弱。

CSS的核心是两个线性反馈移位寄存器（LFSR），一个17位（R1），一个25位（R2），共同构成一个40位的密钥空间。两个LFSR的初始状态由40位的盘密钥（Disk Key）设定。每次两个LFSR会生成9位数据（8位有效位+进位），最终合成1字节的密钥流。LFSR作为伪随机数生成器（PRG），其自身是完全线性的。尽管CSS在最后一步引入了带进位的加法（即非线性元素）来试图混淆结果，但整个系统的核心弱点，正是这个“线性”特征。

CSS的弱点

CSS的系统性弱点主要来自两个层面：

40位短密钥，理论可破：CSS40位密钥的弱点主要来自美国当年严格的密码出口管制。虽然这是其明显弱点，但真正的“杀手锏”是下面基于数学的攻击。

线性与分离，数学瓦解：这是对其破解的真正核心。因为LFSR是线性组件，攻击者可以利用Berlekamp-Massey算法等工具，仅通过一小段已知的密钥流即可重构整个LFSR的线性递归关系，这就是CSS被数学瓦解的根本原因。而两个LFSR的组合方式也使其可以被逐一攻破。

攻击方法

获取初始密钥流：由于DVD内容使用MPEG文件格式，其文件头部有固定的已知明文（如‘MPEG’等特定字段）。将这部分密文与已知明文进行XOR，就可以恢复出最初20字节的CSS密钥流。

暴力破解第一个LFSR：有了20字节的密钥流，攻击者可以遍历所有2¹⁷（约13万）种可能的17位R1的初始状态。对于每种猜测，模拟R1运行，生成前20字节的输出。

验证并同时获取第二个LFSR：将步骤1中获得的真实密钥流减去步骤2中R1的猜测输出。如果R1猜测正确，那么得到的结果正好应该是R2的前20字节输出。

高效验证：检验“推测的R2输出”是否可能由一个25位LFSR生成。由于LFSR的线性特性，这个检测非常简单高效（例如使用Berlekamp-Massey算法）。

一举两得，彻底攻陷：一旦验证成功，不仅R1被破解，R2的初始状态也随之确定。此时，攻击者已经掌握了整个密钥，可以生成后续任意密钥流，解密整部电影。

eStream

eStream 是欧洲密码学卓越网络（ECRYPT）在 2004 年至 2008 年间发起的一个大型项目，旨在为全球学术界和工业界征集、评估并推荐一批全新的、安全且高效的流密码算法，以应对日益增长的信息安全需求。它像是流密码领域的“AES 选拔赛”，但其目标更偏向于探索与推荐。当时，流密码的典型应用场景有两个主要方向，因此 eStream 项目的候选算法也被分为两个 Profile 进行评选：

Profile 1 (面向软件)：追求在通用 CPU（如 Intel 和 AMD 处理器）上达到最高的加密速度和吞吐量。

Profile 2 (面向硬件)：追求在资源受限的硬件设备（如智能卡、嵌入式系统）中，以最低的门电路数量（Gate Count）、功耗和内存实现加解密。

eStream 并非如 AES（高级加密标准）那样要制定唯一的强制标准。它的目标是为开发者提供一组有前途、经过初步验证的新选择。委员会明确指出，这些算法在当时仍较新，可能存在未知的安全风险，未来的分析可能会改变对它们的看法。

流密码的参数不同于之前，同样也有种子，不过还引入了一个新鲜值（nonce），表示主要密钥定下来，永不重复的值（永不重复的唯一值）。PRG需要密钥和新鲜值两个输入，表示为（k，r），这个有序对永不重复。底线是可以重复使用密钥，因为新鲜值确保了有序对唯一，这样我们就不必每次换密钥了。

Salas20

Salsa20，这是一个同时面向硬件和软件设计的流密码。输入128位或256位密钥，这里只看128位版本。输入密钥and随机数，随机数64位，然后产生一个大的输出。

4. PRG定义

两种分布（伪随机分布 vs 真随机均匀分布）

令PRG有密钥空间𝒦，即λ位字符串的集合(安全参数长度 λ = N)，有PRG生成的伪随机分布D0和真随机分布D1，我们的核心目标是定义发生器输出与真正的随机序列无法被区分。

换句话说，需要定义一个分布，从密钥空间里选择密钥时的分布（带R的箭头表示从集合K里均匀的随机选取），这个伪随机字符串的分布与真正的均匀分布没有区别。即安全的本质：伪随机串 ≈ 真随机串，PPT 敌手分不出来。 N位字符串的有限域中，均匀分布会以同等的概率输出每一个字符串，而这个发生器G生成的伪随机分布因为种子空间太小，可能输出结果的集合是全体长串很小很小的一个子集，即N位0,1字符串中一个很小的子集

PRG 统计测试（Statistical Test）𝒜

统计测试𝒜是一个算法，满足输入：任意 l 位二进制串 x，输出一个单位的比特{0, 1} 其中，𝒜(x) = 1 表示测试判定x看起来像随机串，𝒜(x) = 0 表示测试判定x看起来不像随机串。

dan Bonceh老师在课堂上提出了三个实例：0/1均衡性测试（0，1数量应该相近）、二元连续对（00）计数测试（当且仅当串内连续00的总个数接近，若00过多或者过少都会识别为非随即结构，𝒜(x) = 0）、0 的最长游程（最长连续 0 块）测试（ 当且仅当len(substr0)max ≤ 10 × log2(n)时𝒜(x) = 1，全1字符串无连续0，最长游程为0，满足条件，测试会输出1。）

单一统计测试只能捕捉一类特征，存在误判，无法单独作为完整区分器。存在误判可能性，不能完全判定随机性；

区分优势（Advantage）

使用严格的数学语言定义优势（Advantage）：

设：

：输入 PRG 伪随机串时，测试判定 “随机” 的概率；

：输入真随机串时，测试判定 “随机” 的概率。

则定义：测试𝒜针对PRG F的区分优势为：

Adv𝒜, Gprg = |𝒫0 − 𝒫1|

当时 表示𝒫0，𝒫1差距极大，给伪随机串、真随机串时，测试行为完全不一样，𝒜能清晰分开两类分布，该测试破解了PRG；而时 表示𝒫0，𝒫1几乎相当，测试面对伪随机 / 真随机输入表现无差别，无法区分两类串，该测试对G无效。

优势是两类输入下测试阳性概率的差值绝对值；优势越大，区分能力越强；优势可忽略 = 无法区分；常数优势（如 1/6）属于可攻破，说明发生器存在明显偏置漏洞。

安全 PRG 的完整形式化定义

是安全伪随机生成器，当且仅当：对所有有效（PPT，概率多项式时间）统计测试𝒜区分优势Adv𝒜, Gprg(λ)是关于安全参数N = λ的可忽略函数。

两个关键约束解释

只限制「有效统计测试」（PPT算法）：算法运行步数是输入长度的多项式，现实计算机可实现；若去掉 “有效” 限制，不存在任何满足定义的 PRG，即存在指数时间暴力算法遍历所有种子，完美还原种子、100% 区分伪随机输出，但加入PPT限制，定义才具备可满足性，符合现实密码安全模型。

要求对所有有效测试优势均可忽略：安全不能只抵抗某一种统计测试，必须抵抗全部多项式时间区分算法；不存在任何多项式程序能看出PRG输出和真随机的差别。

安全 = 所有多项式时间区分器的优势均可忽略；必须限定 PPT 敌手，否则定义无意义；单一测试无法破解不代表安全，需要抵御全部有效区分测试。

PRG 存在性与复杂度理论（P vs NP 延伸）

目前无法数学严格证明任意一个现实 PRG（AES-CTR、ChaCha 等）是安全的；仅靠多年攻击实践，未找到高效区分器，工程上信任它们。PRG 安全与P ≠ NP等价关联,若存在可严格证明安全的 PRG，则可推出P ≠ NP;若日后证明P = NP，则不存在任何安全 PRG，所有流密码、对称密码根基崩塌；

虽然无法严格证明安全性，但基于数论 / 分组密码构造大量实用 PRG，在工程中广泛使用。

PRG 的不可预测性（下一位不可预测）

不可预测：给定PRG输出串的任意一段前缀，不存在高效算法能以显著高于 1/2 的概率猜出下一个比特。安全PRG一定满足不可预测性。反之：只要存在算法能高效预测下一位，该PRG一定不安全。

姚期智 1982 定理（Yao’s Theorem）

一个比特生成器G是安全PRG（输出分布与均匀分布计算不可区分），当且仅当G满足所有位置下一位不可预测。简写逻辑等价关系： 安全PRG ⇔ 下一位不可预测

正向（之前已证）：安全 → 不可预测 若可预测，则能构造区分器，与安全定义矛盾；

反向（姚定理核心贡献）：不可预测 → 安全 如果不存在任何 PPT 算法能预测任意位置的下一位，那么不存在任何 PPT 统计测试可以区分 PRG 输出与真随机串。

意义：把 “抵抗所有复杂统计测试” 这个很难验证的强条件，简化成更容易分析的 “下一位不可预测” 单一条件，极大降低 PRG 安全性证明难度。

问题：已知该 PRG 存在漏洞 —— 只要给出输出最后一位，就能高效算出第一位。是否说明该发生器可预测？

答：该发生器是可预测的：

存在可利用的结构漏洞 ⇒ G不安全 ⇒ 一定存在某个位置i, 给定前i位可以高效预测第i + 1位

姚定理建立「不可预测性」与「PRG 安全」的等价关系；证明 PRG 安全，只需证明其下一位无法被 PPT 算法预测；只要发现任意结构性区分漏洞，就能直接判定存在下一位预测器。

引入新场景：不再只对比「PRG 输出分布」和「均匀分布」，定义通用任意两个分布𝒫1, 𝒫2是否可区分。

形式化定义：称两个分布𝒫1 ≈ 𝒫2（计算上不可区分），当且仅当对所有 PPT 有效统计测试𝒜，区分优势

是关于安全参数λ的可忽略函数。

即从𝒫1取样交给测试𝒜,同样从𝒫2取样交给测试𝒜两种情况下测试输出 1 的概率几乎没有差别，任何多项式时间算法都找不到稳定区分两者的特征；只有指数时间暴力算法才能看出差异。

5. 流密码安全

攻击者拿到密文后，就算做不到两件极端破坏，系统依然不安全：无法完整还原整个共享密钥；无法完整还原全部明文。

漏洞场景：攻击者不用完全破译密钥 / 全文，只恢复部分明文、单个比特、明文特征，系统就已经不安全。 举例：能算出明文最低一位、能判断明文是 0 还是 1 开头、能区分两段不同明文对应的密文，都属于安全失效。

对比基准：香农完美安全（一次一密 OTP）

攻击者截获密文后，得不到任何关于明文的信息，连明文中任意一个比特都无法预测。

对称加密方案(K, E, D)满足完美安全，当且仅当： 对任意两条等长明文M0, M1 ，密钥均匀随机选取时，加密输出满足：

严格同分布：不管加密M0还是M1密文出现每种取值的概率完全一致；攻击者只看密文，完全无法判断密文来自哪一条明文。

对密钥长度要求极高：密钥长度 ≥ 明文长度，且密钥只能使用一次；普通流密码不可能满足完美安全：流密码仅用短种子（密钥）通过 PRG 生成密钥流，种子空间远小于全部可能密钥流，无法让两条明文的密文严格同分布；只有一次性密码本 OTP 达到完美安全。

完美安全条件太强，无法实用，分两步弱化定义以放宽约束，最终得到适合流密码的语义安全：

第一层弱化：不要求两个密文分布完全相同，仅要求计算上不可区分（PPT 攻击者分不出）；第二层弱化：不对世界上所有明文成立，仅对攻击者主动给出的两条明文M0, M1成立。

核心：对称加密的语义安全（Semantic Security）

语义安全是流密码、分组密码标准安全定义，用区分实验建模，和 PRG 区分游戏逻辑完全统一。

语义安全双实验模型（Exp-0 / Exp-1）

参与方：PPT 攻击者 𝒜 、挑战者 （持有随机密钥 k）统一执行以下流程：

挑战者均匀随机生成加密密钥；攻击者𝒜输出两条等长明文 M0, M1（由攻击者自主选择，代表攻击者关心的明文对）；分支实验：

实验 0 $$：挑战者返回密文 C0 = Ek(M0)

实验 1 $$：挑战者返回密文 C1 = Ek(M1)

攻击者拿到密文后，输出猜测比特b′ ∈ {0, 1}

事件与优势定义

对事件优势做出以下定义：

记实验B中攻击者输出 1的事件为Wb,Pr[W0]为实验0中攻击者输出1的概率，Pr[W1]为实验1中攻击者输出1的概率，攻击者𝒜针对加密方案E的语义安全优势：

Adv𝒜, Esem = |Pr[W0] − Pr[W1]|

当即优势可忽略时看，攻击者在两个实验中行为几乎一样，无法区分加密M0和M1产生的密文；当Adv时不可忽略常熟时，攻击者能稳定分辨，机密系统能够被攻破。

语义安全完整定义

对称加密方案E满足语义安全，当且仅当：对所有 PPT 有效攻击者𝒜，语义安全优势Adv𝒜, Esem 是关于安全参数的可忽略函数。即，不存在任何多项式时间算法，能区分任意两条攻击者选定明文对应的密文。

例题 1：能恢复明文最低位 → 不满足语义安全

场景：加密系统存在漏洞，攻击者拿到密文就能算出明文最后 1 比特。证明该系统不满足语义安全：

构造区分攻击者ℬ：第一步：主动提交两条明文 M0, M1，二者最低位不同（如M0末位0，M1末位1）；拿到密文后调用漏洞算法𝒜算出明文末位；若末位为 0 则输出 0，末位为 1 则输出 1。

计算概率：实验 0（加密M0）：末位恒为 0，攻击者永远输出0 → Pr[W0] = 0;实验 1（加密M1）：末位恒为 1，攻击者永远输出1 → Pr[W1] = 1;

优势：Adv = |0 − 1| = 1,优势为常数 1（极大、不可忽略），系统完全被破解。

关键推论：只要攻击者能学习到明文任意单个比特、任意局部信息，加密方案一定不满足语义安全。这也是流密码安全的底线：不能泄露明文任何比特特征。

例题 2：一次性密码本 OTP 满足语义安全

OTP 加密规则：，密钥 k与明文等长、均匀随机、仅使用一次。

任取等长明文M0, M1：C0 = M0 ⊕ k：k均匀随机，则C0在所有等长比特串上均匀分布；C1 = M1 ⊕ k：k均匀随机，同理C0在所有等长比特串上均匀分布；

两个实验给攻击者的输入密文服从完全相同的均匀分布；无论攻击者是什么 PPT 算法，面对同分布输入，输出 1 的概率必然相等： Pr[W0] = Pr[W1] 即，其优势Adv = 0,对任意攻击者优势恒为 0，因此 OTP 同时满足完美安全 + 语义安全。

安全 PRG 构造的流密码具备语义安全（归约证明全流程）

流密码核心定理：若G是安全PRG，则基于G的流密码E满足语义安全，采用逆否命题 + 密码学标准归约证明：语义安全游戏、PRG 区分优势、计算不可区分、OTP 完美安全。

证明整体思路（归约法，逆否逻辑）

设流密码构造规则：有共享短种子密钥k,调用PRG生成密钥流S = G(k);加密：C = M ⊕ S；解密同理；如果G是安全伪随机生成器，则该流密码E语义安全。

目标：证明「所有 PPT 攻击者𝒜对流密码的于一又是可忽略」。

  1. 反设:存在PPT攻击者𝒜，对流密码有不可忽略的语义优势；
  2. 利用A构造出 PPT PRG 区分器ℬ
  3. 证明ℬ 对 PRGG拥有不可忽略的区分优势；
  4. 与「G 是安全 PRG（所有 PPT 区分器优势可忽略）」矛盾；
  5. 故假设不成立，原定理得证。

两套语义安全游戏定义（证明核心分界）

引入两套完全平行的挑战者，唯一区别是密钥流来源：

伪随机游戏（流密码原生游戏）

密钥流(PRG输出)
